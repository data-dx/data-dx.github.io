InfoRec Software Licensing & Activation Guide

Welcome to InfoRec Software (hereinafter referred to as InfoRec) information logging management tool. To protect your rights regarding current software licensing and subsequent version update downloads, please follow this guide to complete function activation and environment verification registry.

一、 Prerequisites: Download & Installation

  1. Windows Desktop Tool (InfoRec.exe)
    Please visit the project download page https://www.inforec.dev/download.html to download and run InfoRec.exe. This is your core tool for data management, form template printing, and various function verifications.
  2. Verification & Licensing Tools (Blockchain Wallet)
    This system utilizes the Solana blockchain for licensing and environment verification. Please prepare according to your device:
    • Solana Phone Users: Download the "InfoRec App" from the Solana dApp Store.
    • General Desktop/Mobile Users: Install the Phantom or Solflare wallet extension in your browser and authorize via the PWA web version.

二、 Process Description

The system provides two methods: "Online Environment Verification" and "Offline Environment Verification Request":

  1. [Option A: Online Environment Verification (Default)]

    Obtain cloud licensing immediately after successful validation, automatically completing local environment registry.

    • Step 1: Submit Request via Desktop Tool
      Open InfoRec.exe, click "Support > Join Community > Apply for Activation". Fill in the username, contact person, email, and phone number. Select the hardware fingerprint source (PC name or USB sector), then click "Execute". The system will generate a Payment URL (PWA) for regular users or a QR Code for Solana phone users.
    • Step 2: Environment & Technical Verification via Wallet
      Scan the QR Code or access the URL, then choose your intellectual property licensing plan. Confirm in the wallet window and apply SKR transmission verification on the SOL blockchain. This technical verification mechanism is designed to establish a minimum economic barrier (anti-abuse mechanism) and serves as a symbolic reasonable consideration for intellectual property licensing.
    • Step 3: Activate Software License
      Return to the desktop tool, click "Support > Join Community > Execute Environment Verification". The system will automatically compare the on-chain transaction with the environment verification results, completing activation upon validation.
  2. [Option B: Offline Environment Verification Request]

    Applicable to users who cannot operate cryptocurrency directly or whose devices are in offline environments.

    • Step 1: Generate Request File
      Select "Offline License Request" in the function verification window. Fill in the data and click "Execute". The system will automatically generate a request file and guide you to send an email to community assistance personnel.
    • Step 2: Verification via Community Peer Assistance
      If you do not have a designated community assistant, provide relevant proofs to community assistants. The community assistance node will enter "Proxy Mode" and apply SKR transmission verification on the SOL blockchain to transfer and generate the technical license file on your behalf.
    • Step 3: Import License File for Local Registry
      After receiving the "License File" returned by community assistants or verification nodes, save it into the InfoRec program folder. Check "Import File to Activate" on the activation screen, then click "Execute Environment Verification" to complete activation.

三、 Proxy Mode

This mode is used by community assistants to help others complete SKR transfer verification and obtain environment-specific license files:

  1. Community assistants open the verification window and select "Proxy Mode".
  2. Click to read the "License Request File" provided by the applicant.
  3. The community assistant completes the SKR transfer verification via their wallet, and the system generates a "License File" specific to that device and environment.
  4. The community assistant transfers the file to the applicant, who imports it following Step 3 of "Option B".
